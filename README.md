# Hospitality Management Data Analytics Project

The goal is to explore, understand, and analyze transactional hotel data and gradually evolve it into an analytics-oriented system that supports business decision-making.

The project is developed across six sprints using an Agile methodology and includes database exploration, data modeling, advanced SQL querying, cross-database analysis, and dashboard development.

## Project Scope

This project addresses the following areas:

- Database modeling, reverse engineering, and diagramming  
- Writing advanced SQL queries (including analytic functions in later sprints)  
- Working with data across multiple database servers and database types  
- Creating analytic dashboards for non-technical users  
- Applying Agile project management methodology and documentation  

 

## Methodology

The project is completed over six sprints. Each sprint focuses on one or more aspects of the overall system.

- Two-week sprints  
- Approximately three standups per sprint  
- Sprint planning at the start of each sprint  
- Sprint retrospective at the end of each sprint  
- Standup notes, planning notes, and retrospectives are documented as portfolio artifacts  

Standup format:

- What was completed since last standup  
- What will be worked on next  
- Blockers or issues  

A rotating or designated scribe captures notes.
